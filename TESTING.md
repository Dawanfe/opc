# 邀请码功能自动化测试文档

## 📋 测试方案概述

我们为邀请码功能提供了三种测试方案：

### 1. **集成测试**（推荐用于快速验证）
- 文件: `tests/invite.test.ts`
- 测试范围: 完整的端到端功能测试
- 测试内容: API 调用、数据库操作、业务逻辑验证

### 2. **单元测试**（使用 Jest）
- 文件: `tests/invite-utils.test.ts`
- 测试范围: 工具函数的单元测试
- 测试内容: 邀请码生成、格式验证等纯函数

### 3. **性能测试**
- 文件: `tests/invite-performance.test.ts`
- 测试范围: 性能和压力测试
- 测试内容: 吞吐量、并发处理、查询性能

---

## 🚀 快速开始

### 方式一：使用测试脚本（最简单）

```bash
cd next
chmod +x test-invite.sh
./test-invite.sh
```

这个脚本会自动：
- 检查并启动开发服务器（如果未运行）
- 初始化数据库
- 运行所有集成测试
- 清理测试环境

### 方式二：手动运行集成测试

```bash
# 1. 启动开发服务器（新终端）
cd next
npm run dev

# 2. 运行测试（另一个终端）
cd next
npx tsx tests/invite.test.ts
```

### 方式三：运行单元测试（Jest）

```bash
cd next/tests
npm install
npm test
```

### 方式四：运行性能测试

```bash
cd next
npx tsx tests/invite-performance.test.ts
```

## 🐛 故障排查

### 问题1: 测试失败 "开发服务器未运行"

**解决方案:**
```bash
# 手动启动开发服务器
cd next
npm run dev
```

### 问题2: 测试失败 "该手机号已注册"

**说明:** 这是正常的，表示测试数据已存在。测试会自动跳过。

**清理方法:**
```bash
# 删除测试数据库或清理测试用户
cd next
npm run db:init  # 重新初始化数据库
```

### 问题3: 数据库锁定错误

**解决方案:**
```bash
# 停止所有使用数据库的进程
pkill -f "node.*next"
rm -f next/data/opc.db-shm next/data/opc.db-wal
```

### 问题4: Jest 单元测试失败

**解决方案:**
```bash
cd next/tests
npm install
npm test -- --verbose
```

## 📈 持续集成（CI）

### GitHub Actions 配置示例

```yaml
name: Invite Feature Tests

on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v3

      - name: Setup Node.js
        uses: actions/setup-node@v3
        with:
          node-version: '18'

      - name: Install dependencies
        run: |
          cd next
          npm install

      - name: Initialize database
        run: |
          cd next
          npm run db:init

      - name: Run integration tests
        run: |
          cd next
          npm run dev &
          sleep 10
          npx tsx tests/invite.test.ts

      - name: Run unit tests
        run: |
          cd next/tests
          npm install
          npm test
```
